# 深澤式品質検証プロンプト（月次報告書 A4/A3 印刷レイアウト調整）

あなたは極めて厳格なQAエンジニアです。
以下の前提条件と検証手順に従い、月次報告書の印刷（PDF出力）時にレイアウトが左右にはみ出ず、A4およびA3用紙サイズに応じて最適に印刷されるかを検証してください。

---

## 1. 検証対象と要件定義

### 【項目A】@page サイズ指定の動的切り替えと用紙最適化
* **挙動要件:**
  1. `print_report.html` 上部のツールバーにおいて、「A4サイズで印刷」と「A3サイズで印刷」ボタンがそれぞれ正常に機能すること。
  2. ボタンクリック時、JavaScript によって `<style id="dynamic-page-style">` 内に適切な `@page` ルール（A4なら `size: A4 portrait; margin: 12mm 10mm 12mm 10mm;`、A3なら `size: A3 portrait; margin: 15mm 12mm 15mm 12mm;`）が動的に注入され、`body` に対応するクラス（`print-size-a4` または `print-size-a3`）が正しく設定された上で印刷ダイアログが表示されること。

### 【項目B】印刷時のテーブルあふれ・文字はみ出し防止
* **挙動要件:**
  1. 物件名や製品名など、テキストが長くなっても詳細データ一覧テーブル (`detail-table`) が印刷範囲の右端を突き破ってはみ出さないこと。
  2. テーブルに `table-layout: fixed` が強制適用され、長文に対して `word-break: break-all` と `white-space: normal` による自動折り返しが効いていること。
  3. 列幅比率がA4/A3それぞれで適切に定義されており、特にA4用紙印刷時にも文字が他の列に被ったり隠れたりしないこと。

### 【項目C】用紙サイズに応じたレスポンシブ印刷レイアウト (KPIカード・在庫金額グラフ)
* **挙動要件:**
  1. **KPIカード (.kpi-grid):**
     - A4/A3用紙のいずれにおいても、3列の並びが維持されること。
     - A4用紙印刷時にはフォントサイズおよびパディングが縮小され、3列並びでもカード内の文字があふれないこと。
  2. **在庫評価分析 (.inventory-grid):**
     - A4用紙印刷時には、2つのグラフセクションが `grid-template-columns: 1fr` によって「縦並び」にスタックされ、各グラフ幅が100%になることで、はみ出しが完全に防止されていること。
     - A3用紙印刷時には、十分な幅があるため `grid-template-columns: 1fr 1fr` により「横2列並び」で美しく配置されること。
  3. **部門別コスト配分 (.dept-grid):**
     - 印刷時には 3列並びが適用され、各カードが適切に収まること。

### 【項目D】バージョンとビルドの整合性
* **挙動要件:**
  1. バージョン表記が `v5.10` になっており、アセット読み込みURLのキャッシュバスターが `v=20260520_9` に更新されていること。
  2. `mobile.html` がビルドされ、最新の `app-mobile.js` 内のクラス名追加等の修正が正しくインライン化されて反映されていること。

---

## 2. 厳格な検証手順

### ステップ1: コード論理の静的検証
1. `print_report.html` に定義された印刷用スタイルが、インラインスタイル（例：`#report-print-area` の `max-width: 1000px`）に勝つために `!important` で上書きを行っているか？
2. 動的切り替え関数 `setupPageSize` および `setPageSizeAndPrint` がバグ（シンタックスエラーやヌルポインター）なく動作するか？
3. `app.js` と `app-mobile.js` から生成される HTML が、それぞれ `print_report.html` の `@media print` にて定義されたクラス名（`kpi-grid`、`inventory-grid`、`detail-table`、`dept-grid`）と完全に一致しているか？

### ステップ2: モバイルビルド整合性検証
1. `mobile.html` 内を検索し、最新の `renderReport` HTML内のクラス名追加（例：`kpi-grid` や `inventory-grid`）が正しくビルドに含まれているか？

### ステップ3: テスト実行（モック等による静的確認またはデバッグ）
1. 開発環境のブラウザ印刷シミュレーションやレンダリング結果の確認。
2. 変更ファイルをすべて Git にコミットし、`git push` が成功することを確認する。
